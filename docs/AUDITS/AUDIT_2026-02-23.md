## Detailed Findings

### [HIGH] #1: No HTTP Status Validation in API Clients

**Location:** `src/lib/apis/snowtrace.ts:13`, `src/lib/apis/routescan.ts:17,33`
**Risk:** If Snowtrace/Routescan returns 503 or HTML error page, `res.json()` throws unhandled error
**Impact:** Dashboard page crashes with unhandled rejection
**Verified:** Yes — code reads `res.json()` without checking `res.ok`
**Fix:** Add status check before parsing:
```typescript
if (!res.ok) {
  return null; // or throw new Error(`API error: ${res.status}`)
}
const json = await res.json();
```
**Status:** OPEN

### [HIGH] #2: No Fetch Timeout in API Clients

**Location:** `src/lib/apis/snowtrace.ts:10-12`, `src/lib/apis/routescan.ts:16,32`
**Risk:** If external API hangs, request blocks indefinitely, server component times out
**Impact:** Users see infinite loading, Vercel function timeout (10s default)
**Verified:** Yes — no `AbortController` or timeout configuration
**Fix:** Add AbortController with 10s timeout:
```typescript
const controller = new AbortController();
const timeout = setTimeout(() => controller.abort(), 10000);
const res = await fetch(url, { signal: controller.signal });
clearTimeout(timeout);
```
**Status:** OPEN

### [HIGH] #3: No Rate Limit Protection for External APIs

**Location:** `src/lib/apis/snowtrace.ts`, `src/lib/apis/routescan.ts`
**Risk:** Concurrent requests (3x parallel in `Promise.all`) can exceed 2 req/sec limit
**Impact:** API returns 429, all data loading fails
**Verified:** Yes — `analytics/route.ts:44` calls `getContractInfo` + `getTransactions` + `getFirstTransaction` in `Promise.all`; `getContractInfo` internally calls `getContractSource` + `getContractABI` sequentially = 4 total requests to 2 APIs
**Fix:** Sequential calls for same API, or add request queue with 500ms delay
**Status:** OPEN

### [MEDIUM] #4: Wrong Cache TTL for First Transaction

**Location:** `src/lib/apis/routescan.ts:36`
**Risk:** First transaction cached for 24 hours (`CACHE_TTL.ABI`) instead of appropriate TTL
**Impact:** Minor — first tx data (creation date) rarely changes, but semantically wrong
**Verified:** Yes — line 36 uses `CACHE_TTL.ABI` (24h) for first transaction data
**Fix:** Change to `CACHE_TTL.CONTRACT_INFO` (5 min) or create dedicated TTL
**Status:** OPEN

### [MEDIUM] #5: No Error Feedback on Period Change Failure

**Location:** `src/components/dashboard/TabNavigation.tsx:34-39`
**Risk:** If API returns `{ success: false }`, user sees stale data with no indication of failure
**Impact:** Silent failures confuse users
**Verified:** Yes — line 35 checks `json.success` but does nothing on failure; catch only logs to console
**Fix:** Add error state and show message to user
**Status:** OPEN

### [MEDIUM] #6: NaN Propagation in Format Functions

**Location:** `src/lib/utils.ts:34-36`, `src/lib/processing/analytics.ts:13,23`
**Risk:** If `parseInt(tx.timeStamp)` returns NaN, it propagates through calculations, displaying "NaN%" in UI
**Impact:** Broken UI for malformed API responses
**Verified:** Yes — `formatPercentage()` calls `value.toFixed(1)` without NaN guard; `parseInt()` on undefined/empty string returns NaN
**Fix:** Add NaN guard in `formatPercentage` and other format functions:
```typescript
export function formatPercentage(value: number): string {
  if (isNaN(value)) return '0.0%';
  return value.toFixed(1) + '%';
}
```
**Status:** OPEN

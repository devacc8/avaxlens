## Detailed Findings

### [MEDIUM] #1: rpcCall in useNetworkStats lacks response validation

**Location:** `src/lib/hooks/useNetworkStats.ts:16-23`
**Risk:** If RPC returns malformed response (no `result` field), `json.result` is undefined, `parseInt(undefined, 16)` returns `NaN`, displayed as "NaN nAVAX"
**Impact:** UI shows broken data to users
**Fix:** Add null/type check before parseInt:
```typescript
async function rpcCall(method: string): Promise<string | null> {
  const res = await fetch(AVAX_RPC, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ jsonrpc: '2.0', id: 1, method, params: [] }),
  });
  const json = await res.json();
  if (typeof json.result !== 'string') return null;
  return json.result;
}
```
**Verified:** Yes — read source, confirmed no type check on `json.result`
**Status:** FIXED (v0.5.4)

### [MEDIUM] #2: localStorage data not validated on read

**Location:** `src/lib/recent-searches.ts:15`
**Risk:** If localStorage is manually tampered (or corrupted), `JSON.parse(raw) as RecentSearch[]` trusts the data shape. Rendering unvalidated `name` field in JSX could display unexpected content (though React auto-escapes, so no XSS).
**Impact:** Broken UI if data shape is wrong (missing fields → runtime error in ContractExplorer)
**Fix:** Add basic shape validation:
```typescript
const parsed = JSON.parse(raw);
if (!Array.isArray(parsed)) return [];
return parsed.filter(
  (r: unknown): r is RecentSearch =>
    typeof r === 'object' && r !== null &&
    typeof (r as RecentSearch).address === 'string' &&
    typeof (r as RecentSearch).name === 'string'
);
```
**Verified:** Yes — read source, confirmed no validation after JSON.parse
**Status:** FIXED (v0.5.4)

### [LOW] #3: No timeout on DeFiLlama/RPC fetch calls

**Location:** `src/lib/hooks/useNetworkStats.ts:16,28-29`
**Risk:** If DeFiLlama or Avalanche RPC is slow/hanging, fetch waits indefinitely (browser default timeout ~300s). React Query retry will stack requests.
**Impact:** Slow landing page load, potential memory leak from stacked requests
**Fix:** Add AbortSignal.timeout:
```typescript
fetch(AVAX_RPC, { ..., signal: AbortSignal.timeout(10_000) })
fetch(DEFILLAMA_PRICE, { signal: AbortSignal.timeout(10_000) })
fetch(DEFILLAMA_CHAINS, { signal: AbortSignal.timeout(10_000) })
```
**Verified:** Yes — read source, no timeout on any of the 3 fetch calls
**Status:** FIXED (v0.5.4)

### [LOW] #5: React Query retry on network stats may spam APIs

**Location:** `src/components/providers/QueryProvider.tsx:12`
**Risk:** Default retry: 1, combined with refetchInterval: 30s on network-stats. If DeFiLlama rate limits, each failed request triggers 1 retry = 2 requests per 30s. Not critical but wastes client resources.
**Impact:** Minor — extra failed requests
**Fix:** Set retry: 0 for network-stats query, or add `retry: false` in useNetworkStats options
**Verified:** Yes — QueryProvider sets retry: 1 globally, useNetworkStats doesn't override
**Status:** FIXED (v0.5.4)

### [INFO] #7: AbortSignal.timeout browser support

**Location:** `src/lib/hooks/useAnalytics.ts:8`, `src/lib/hooks/useTransactions.ts:9`
**Risk:** `AbortSignal.timeout()` not supported in Safari <16.4 (released March 2023). Most users should be fine, but older Safari/iOS may error.
**Impact:** Low — modern browsers only affected
**Recommendation:** Document minimum browser requirement or add polyfill
**Status:** ACCEPTABLE

### [INFO] #8: CSP allows unsafe-inline and unsafe-eval

**Location:** `next.config.ts:12`
**Risk:** `unsafe-inline` and `unsafe-eval` in script-src weaken CSP. However, these are required by Next.js runtime (webpack, hot reload, inline scripts).
**Impact:** Standard Next.js limitation — not fixable without nonce-based CSP
**Recommendation:** Track Next.js CSP nonce support (experimental in v15+)
**Status:** ACCEPTABLE — Next.js requirement

### [INFO] #9: Rate limiter unbounded in-memory Map

**Location:** `src/lib/rate-limit.ts:4`
**Risk:** The `hits` Map grows with unique IPs. Cleanup runs every 5 minutes but only removes expired entries. On Vercel serverless, each instance has its own Map (resets per cold start), so this is not a real concern.
**Impact:** None on Vercel (serverless). Would matter on long-running server.
**Recommendation:** No action needed for current deployment
**Status:** ACCEPTABLE
